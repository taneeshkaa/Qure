Project: Qure (Admin Control & Patient Attachments)Scope: Global Management & Patient File UploadsStack: Node.js, Express, Neon DB (PostgreSQL), Cloudinary/AWS S3 (for file storage)1. Feature 1: The Master Admin DashboardThe Admin serves as the "System Auditor." This role requires high-privilege access to the entire relational tree in Neon DB.A. Global "Delete" Logic (Cascade vs. Soft Delete)To maintain database integrity, we will implement Soft Deletes (adding a deleted_at timestamp) instead of hard deleting records. This prevents breaking historical appointment data.Delete Hospital: Automatically flags associated Doctors and Pharmacies as "Inactive."Delete Doctor/Pharmacy: Removes them from the search index (Phase 2) so patients can no longer book them.Delete Patient: Revokes login access and hides their medical profile from the search.B. Advanced Admin Features (Suggested)System Analytics: A query-heavy endpoint to fetch "Total Appointments Today" and "Top Performing Hospitals."Specialty Management: Ability to add new keywords to the Symptom-to-Specialty mapping (e.g., adding "Monkeypox" to "Infectious Diseases").Verification Toggle: Admin must "Approve" a hospital after registration before it goes live in the search results.2. Feature 2: Patient File Upload SystemDuring the appointment booking process (Phase 2), patients now have the option to attach digital evidence of their medical history.A. Upload LogicSince Neon DB is a relational database, we do not store the actual images in the database (this would make it slow and expensive).Storage: Use a cloud provider (e.g., Cloudinary or S3).Reference: Store the URL and File Metadata in a new table in Neon.B. Data CategorizationThe backend will expect an array of files categorized by the user:TEST_RESULT: Blood reports, X-rays, etc.OLD_PRESCRIPTION: Handwritten notes from previous doctors.MEDICINE_IMAGE: Photos of medicine strips where the name is unknown.3. API Endpoints (Phase 4)MethodEndpointDescriptionDELETE/api/v1/admin/entity/:type/:idSoft-deletes a Hospital, Doctor, Patient, or Pharmacy.GET/api/v1/admin/statsAggregated data for the dashboard (Total users, bookings, etc.).POST/api/v1/patient/uploadMulter middleware handles the file stream to the cloud; returns URLs to be saved in Neon.GET/api/v1/doctor/attachments/:apptIdAllows doctors (Phase 3) to view the uploaded files of the patient sitting in front of them.5. Security & Error HandlingFile Validation: The Express backend must validate file size (max 5MB) and type (PDF/JPG/PNG only) before sending to the cloud to prevent server bloat.Privilege Escalation: Every Admin endpoint must have a verifyAdmin middleware to check JWT roles, ensuring a patient cannot accidentally delete a hospital.Orphaned Files: Logic to delete the cloud file if the patient cancels the appointment or the record is deleted.
