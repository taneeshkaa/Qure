Design Document: Qure Frontend (Phase 1)Project: Qure (Powered by Antigravity)Stack: React.js, Tailwind CSS, Framer Motion (for Lusion-style animations), Three.js (optional for background depth).Theme: Dark Mode / Glassmorphism / Fluid Motion.1. Visual Strategy: "The Lusion Aesthetic"Instead of static boxes, we will use:Immersive Backgrounds: A subtle 3D grain or fluid gradient background that reacts to mouse movement.Micro-Interactions: As the user typing, the text "levitates" into place.Step-based Navigation: To prevent cognitive overload, we use a "Progressive Disclosure" model (one question at a time or small logical clusters).2. Component ArchitectureA. The "Identity Toggle" (Hospital vs. Patient)A high-contrast, animated switch. Selecting "Hospital" shifts the color palette (e.g., Deep Emerald), while "Patient" shifts to a calm "Electric Blue."B. Hospital Registration Flow (The Data Collector)Smart Dropdowns: Custom-built searchable selects for State > City > Hospital.Logic: Selecting a State triggers a fetch to your Node/Express backend to populate the City list instantly.The "Staff Roster" Matrix: A dynamic list where the user can "Add Doctor." Each entry contains:Doctor Name (Input)Specialty (Select/Searchable)Ownership Details: Minimalist inputs for Owner Name and Contact Person.Dual Phone Column: A split-input component where the second field is visually dimmed to indicate it is optional.C. Patient Registration Flow (The Persona)Basic Identity: Floating label inputs for Name and Email.Medical Essentials: A grid-based selection for Blood Group and Gender.History Text Area: A clean, expandable "Condition Notes" box with a character counter.Emergency Contact: A separate, visually grouped "Safe-Zone" section for Emergency details.3. Interaction & Animation Logic (Framer Motion)Entrance: Components should not just "appear"; they should slide up with a spring effect ($y: 20 \to 0$, $opacity: 0 \to 1$).Validation Shakes: If a mandatory field is missed, the field performs a subtle horizontal shake rather than a standard red text alert.Loading States: While the backend (Neon DB) is processing, use a "Pulse" animation on the submit button that mimics a heartbeat.4. Technical Integration (Frontend to Backend)State Management: React Context or Zustand to hold the form data as the user moves through steps.API Layer: Axios instances configured to point to your Node/Express endpoints.Form Handling: React Hook Form combined with Zod for schema validation (matching your backend constraints).5.
