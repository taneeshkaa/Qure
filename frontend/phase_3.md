Design Document: Qure Frontend (Phase 3)Project: Qure (Clinical & Pharmacy Operations)Stack: React.js, Tailwind CSS, Framer Motion, Socket.io-client (for real-time prescription alerts).Concept: "The Accelerated Cure"1. Feature 1: The Doctor’s Window (The Consultation Hub)This view is designed for speed and clarity while the patient is physically present.A. The Patient "Live Card"Trigger: When the doctor clicks "Call Next Patient" or selects a Token Number (e.g., #24), the Patient Card slides into view.Content: * Registration Stats: Age, Gender, Blood Group, Allergies, and Chronic Diseases.Booking Context: The specific problem the patient described when booking.Visual Attachments: A gallery of the images/PDFs the patient uploaded (Phase 4) like old prescriptions or test results.Logic: The doctor gets a 360-degree view without asking the patient to repeat their history.B. The Prescription InputThe Smart Textbox: A minimalist, focused text area where the doctor types medicines and tests.Action: A "Send to Chemist" button that triggers a backend event.Logic: Upon sending, the UI provides a "Sent" confirmation, and the current token is marked as "Completed" in the doctor's queue.2. Feature 2: The Chemist’s Window (Advance Packing)This interface is for the pharmacy staff to prepare orders before the patient even leaves the doctor's cabin.A. The Incoming Prescription FeedReal-time Queue: As soon as a doctor hits "Send," a new card appears on the Chemist's screen via WebSockets.Initial View (Privacy Mode): Only the Token Number and the Prescription Text are visible.Workflow Status: Staff can toggle a status from "Pending" to "Packing" to "Ready."B. The "Anti-Scam" Handover ProtocolThe Identity Lock: To prevent scammers from claiming random medicine envelopes, the patient's sensitive data is hidden behind a "Verify" button.Verification Steps:Patient arrives and provides their Token Number.Chemist clicks the Token on the screen.The UI reveals the Full Name and Phone Number.Chemist asks: "What is your name?" and matches it with the screen before handing over the medicine.3. Interaction & UI LogicA. Real-Time Sync (Socket.io)We will use WebSockets so the Chemist doesn't have to refresh the page. New prescriptions "pop" onto the top of the list instantly.B. Responsive LayoutsDoctor View: Optimized for desktops/tablets with a side-by-side view (Patient Card on the left, Prescription Box on the right).Chemist View: A "Kanban" style board (Pending | Processing | Ready) for easy management of multiple envelopes.4. Tech Stack & Integration SummaryFrontend: React.js with react-toastify for real-time notifications when a new prescription arrives.State Management: Zustand to manage the active patient session for the doctor.Logic Flow:Doctor: Views Card $\rightarrow$ Types Prescription $\rightarrow$ Hits "Send."Backend: Updates Neon DB status; emits socket event to the specific Hospital's Chemist.Chemist: Receives alert $\rightarrow$ Packs envelope $\rightarrow$ Sets to "Ready."Handover: Patient gives Token $\rightarrow$ Chemist verifies Name/Phone $\rightarrow$ Completed.5. Error HandlingNetwork Interruption: If the connection to the chemist drops, the Doctor's UI will show a "Draft Saved Locally" warning until the connection is restored.Double Verification: If the name provided by the patient doesn't match the screen, the UI flags the entry in red to alert the staff.
